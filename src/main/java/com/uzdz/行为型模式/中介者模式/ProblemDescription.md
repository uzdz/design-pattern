## 中介者模式（Mediator Pattern）

### 模式概述

> 用一个中介对象来封装一系列的对象交互，中介者使各对象不需要显式地相互引用，从而使其耦合松散，而且可以独立地改变它们之间的交互。

### 优点
1. 通过将对象彼此解耦，可以增加对象的复用性。
2. 通过将控制逻辑集中，可以简化系统逻辑。
3. 可以让对象之间所传递消息变得简单而大幅减少维护成本
4. 提高系统的灵活性，使得系统易于扩展和维护

### 缺点
1. 中介者类承担了较多的责任，一旦中介者出现了问题，整了系统都会收到影响(双刃剑)。
2. 如何设计不当，中介者对象本身会变得过于复杂。

## 注意事项
1. 由于使用中介者模式，将对象间的通信集中在中介者身上，势必会增加中介者的复杂度
2. 由于所有对象间的通信都由中介者协调，那么中介者的高可用就显得格外重要。
